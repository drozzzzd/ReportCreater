# ReportCreater

**ReportCreater** - настольный конструктор TXT-отчетов для QA, поддержки и ручной проверки ошибок. Программа помогает быстро собрать структуру отчета, не потерять черновик, подготовить вложения и сохранить итоговый файл в едином формате.

## Что умеет программа

- Создает отчет из нескольких разделов с отдельными полями для предусловия, положения, вложений и результата проверки.
- Позволяет вручную указать номер ошибки для каждого раздела и подтверждает правильность номера перед сохранением.
- Поддерживает типы записи **Error**, **Question** и **Text**. Режим **Text** выводит введенный текст напрямую, без служебной нумерации вида `# 1 -`.
- Показывает живой предпросмотр итогового TXT-отчета с переносом длинных строк внутри окна.
- Сохраняет отчет с именем по шаблону `Дата-время-SIR.txt`. Если поле `SIR` не заполнено, сохранение блокируется предупреждением.
- Открывает сохраненный отчет из папки и подтягивает его данные обратно в поля программы.
- Автоматически сохраняет локальный черновик после действий пользователя, чтобы данные не пропали при случайном закрытии.
- Позволяет вручную сбросить autosave-черновик через отдельную опасную кнопку с подтверждением.
- Делит интерфейс на вкладки: **Конструктор отчетов** с параметрами и **Заполнение ошибок и предпросмотр** с рабочей областью.
- Сворачивает блоки конструктора и параметров, чтобы освободить место для работы.
- Автоматически расширяет текстовые поля, когда пользователь вводит большой текст.
- Хранит локальные отчеты в папке `reports`, которая не отправляется в Git.

## Параметры отчета

В параметрах можно заполнить:

- `Build`
- `BD`
- `SQL`
- `DOC`
- `Tax Referens`
- `SIR`
- исполнитель
- папку сохранения отчета

Поле `SIR` является служебным: оно используется для имени файла отчета и генерации вложений, но не выводится отдельной строкой в сам отчет.

## Вложения

Кнопка **Вложение** формирует строку attachment на основе `SIR`, исполнителя и счетчика внутри текущей сессии. Счетчик увеличивается последовательно, поэтому вложения получают понятные номера вроде `_01`, `_02`, `_03`.

## Справка ФНС

Кнопка **Справка ФНС** использует значение поля `Tax Referens` и помогает вставлять готовые пути по разделу документов:

- `Журнал сформированных справок для налоговых`
- `Отчет по сформ.справкам в старой версии программы`
- `Отчет по предоставленным справкам`
- `Справка для налоговой`

У каждого пункта есть действие **Просмотр**, которое вставляет полный путь с нужными промежуточными шагами по правилам заполнения в программе.

```bash

```
