Зависимости вкладок ReportCreater для ChatGPT
Дата анализа: 2026-04-28
Проект: c:\divan_api_tester

Важно: promt.md на диске пустой, размер 0 байт. Поэтому этот файл фиксирует вторую просьбу: зависимости по открываемым вкладкам и их логике в программе.

1. Точка входа приложения

main.py:
- Загружает config.json через load_config().
- Создает QApplication.
- Создает главное окно ReportsWindow из mod/reports/reports_window.py.
- Передает в ReportsWindow конфиг и путь к config.json.

config.json:
- general.reports_dir задает базовую папку отчетов.
- По умолчанию используется reports.

Главная зависимость UI:
- main.py -> mod.reports.reports_window.ReportsWindow -> mod/reports/ui/reports_window.py.

Внешняя библиотека:
- PyQt6 - все виджеты интерфейса, меню, вкладки, диалоги, иконки, открытие папки.

2. Главные вкладки окна

Главное окно создает QTabWidget self.main_tabs в ReportsWindow._build_ui().
В программе есть две главные вкладки:
- Конструктор отчетов.
- Заполнение ошибок и предпросмотр.

2.1. Вкладка "Конструктор отчетов"

Создается в:
- mod/reports/ui/reports_window.py -> ReportsWindow._build_setup_tab().

Что содержит:
- Scroll area с рабочим содержимым.
- Верхнюю панель действий.
- Hero-панель.
- Группу параметров отчета.

Кнопки верхней панели:
- "+ Раздел" -> ReportsWindow.add_section().
- "- Последний" -> ReportsWindow.remove_last_section().
- "Сохранить отчет" -> ReportsWindow.save_report().
- "Открыть отчет" -> ReportsWindow.select_report_file().
- "Сброс черновика" -> ReportsWindow.reset_autosave_draft().
- Счетчик "Разделов: N" обновляется в ReportsWindow.renumber_sections().

Поля параметров отчета:
- Build.
- BD.
- SQL, включается или выключается чекбоксом include_sql.
- DOC, включается или выключается чекбоксом include_doc.
- Tax Referens, включается или выключается чекбоксом include_tax_reference.
- Папка сохранения.
- SIR.
- Исполнитель.
- Следующее вложение.

Как данные вкладки уходят в логику:
- ReportsWindow.collect_metadata() собирает поля в ReportMetadata.
- ReportMetadata используется для предпросмотра, сохранения, валидации, autosave и генерации вложений.
- SIR нужен для имени файла отчета и attachment-строк.
- Исполнитель нужен для attachment-строк.
- SQL, DOC и Tax Referens попадают в итоговый TXT только если включены соответствующие чекбоксы.

Живые обновления:
- ReportsWindow._connect_live_updates() подписывает поля на refresh_live_state().
- При изменении любого поля обновляются:
  - предпросмотр;
  - подсказка следующего attachment;
  - прогресс заполненности;
  - autosave-кеш.

Файловые зависимости вкладки:
- mod/reports/ui/reports_window.py - UI и обработчики.
- mod/reports/core/report_data.py - структура ReportMetadata.
- mod/reports/core/report_builder.py - сборка preview и сохранение TXT.
- mod/reports/core/report_parser.py - восстановление данных при открытии TXT.
- mod/reports/core/attachment_counter.py - расчет следующего attachment.
- mod/reports/dialogs/validation_issues.py - диалог ошибок валидации.
- mod/reports/utils/widget_helpers.py - clearable context menu и invalid-state.
- styles/reports_window.qss - стили вкладки.

2.2. Вкладка "Заполнение ошибок и предпросмотр"

Создается в:
- mod/reports/ui/reports_window.py -> ReportsWindow._build_fill_tab().

Что содержит:
- QSplitter content_splitter.
- Левая часть: список секций отчета.
- Правая часть: предпросмотр итогового TXT и прогресс заполненности.

Левая часть:
- sections_panel.
- sections_scroll_area.
- sections_layout.
- Каждый раздел является ReportSectionWidget из mod/reports/ui/section_widget.py.

Правая часть:
- preview_text - readonly QPlainTextEdit с итоговым текстом отчета.
- completion_bar - QProgressBar заполненности.
- completion_label и completion_hint.

Как данные вкладки уходят в логику:
- ReportsWindow.collect_sections() собирает все ReportSectionWidget в список ReportSectionData.
- ReportsWindow.build_preview_text() передает ReportMetadata и список ReportSectionData в TextReportBuilder.build_content().
- refresh_live_state() пишет результат build_preview_text() в preview_text.
- save_report() сохраняет тот же набор данных в TXT через TextReportBuilder.save().

Адаптивность:
- ReportsWindow._update_responsive_layout() меняет ориентацию content_splitter:
  - ширина меньше 1040: вертикально;
  - иначе горизонтально.
- ReportsWindow._update_density_mode() меняет плотность UI секций и ширины кнопок.

Файловые зависимости вкладки:
- mod/reports/ui/reports_window.py - вкладка, splitter, preview, progress, сбор данных.
- mod/reports/ui/section_widget.py - секции и поля внутри секций.
- mod/reports/core/report_data.py - ReportSectionData.
- mod/reports/core/report_builder.py - итоговый TXT и preview.
- mod/reports/core/attachment_counter.py - attachment-подсказки.
- mod/reports/widgets/smooth_scroll_area.py - плавная прокрутка.
- mod/reports/widgets/resizable_text_edit.py - авторазмер текстовых полей.
- mod/reports/widgets/segmented_control.py - переключатель Error / Question / Текст.
- styles/reports_window.qss - стили вкладки.

3. Логика одного раздела отчета

Создается в:
- mod/reports/ui/section_widget.py -> ReportSectionWidget.

Раздел содержит:
- Номер ошибки.
- Название.
- Блок Pre-Condition.
- Блок Scenario.
- Блок Ошибка / Вопрос / Текст.
- Переключатель типа: Error, Question, Текст.

Mode-кнопки и кнопка "Справка ФНС":
- Добавляются только в блоки Pre-Condition и Scenario.
- В блоке Ошибка / Вопрос / Текст mode-кнопок нет, там только helper-кнопки и тип записи.

Helper-кнопки:
- Rus -> вставляет "' '".
- Прочерк -> вставляет " - ".
- Вложение -> вызывает attachment_provider из ReportsWindow.build_attachment_text().
- -вложение -> удаляет последнюю строку, начинающуюся с attachment:.

Как раздел отдает данные:
- ReportSectionWidget.get_section_data() возвращает ReportSectionData.
- content_changed сигналит в ReportsWindow.refresh_live_state().

Зависимости раздела:
- ReportSectionWidget -> MenuCatalog -> menu_catalog.json.
- ReportSectionWidget -> MenuFormatter.
- ReportSectionWidget -> MenuBuilder.
- ReportSectionWidget -> ReportsWindow.build_attachment_text().
- ReportSectionWidget -> ReportsWindow.build_tax_reference_menu_prefix().
- ReportSectionWidget -> FlowLayout, ResizablePlainTextEdit, IssueTypeSegmentedControl.

7. Особенности каталога меню

Книга записи на прием:
- Использует appointment.menu_names и appointment.items.
- У этого режима нет отдельного словаря tabs.
- Каждый menu_name становится tabmenu.

Пациенты:
- patient.menu_names задает верхний список разделов.
- patient.tabs задает разделы, у которых есть вложенные вкладки.
- patient.nested_items задает пункты внутри пары раздел + вкладка.
- patient.submenu_items задает пункты для разделов без вложенных tabs.
- patient.tab_mapping используется для форматирования разделов без tabs.
- search_view_items автоматически добавляются в каждый patient.submenu_items после удаления финальной "Отмена".

Практика:
- practice.items задает прямые пункты раздела.
- practice.tabs задает вкладки внутри раздела.
- practice.nested_items задает пункты внутри пары раздел + вкладка.
- Если у раздела есть и direct_items, и tabs, MenuBuilder ставит separator между ними.

Маркетинг, Страхование, Настройки, Лаборатории:
- Работают через общий шаблон _build_simple_tabbed_menu().
- Если для раздела есть tabs, клик по разделу вставляет строку раздела, стрелка открывает вкладки.
- Если tabs нет, раздел является конечным пунктом.

9. Сборка итогового TXT

Основной класс:
- mod/reports/core/report_builder.py -> TextReportBuilder.

Вход:
- ReportMetadata из collect_metadata().
- list[ReportSectionData] из collect_sections().

Порядок верхних строк:
- Build <build>.
- BD - <database>.
- SQL - <sql_value>, если include_sql true.
- DOC - <doc_value>, если include_doc true.
- Tax Referens - <tax_reference_value>, если include_tax_reference true.

SIR:
- Не пишется отдельной строкой в тело отчета.
- Используется в имени файла при save().
- Используется в attachment-строках.

Обычный раздел:
- # <number> - <title>
- Pre-Condition:
- строки precondition
- Scenario:
- строки scenario
- <Error или Question>: <issue_text>

Текстовый раздел:
- Если issue_type == "Текст", выводится только issue_text.
- Номер, заголовок, Pre-Condition и Scenario для такого блока не выводятся.

Пустой раздел:
- Не попадает в итоговый TXT.

Имя файла:
- TextReportBuilder.save() пишет в output_dir файл формата YYYY-MM-DD_HH-MM-<SIR>.txt.
- Недопустимые символы в SIR заменяются на "_".

10. Открытие сохраненного отчета

Кнопка:
- "Открыть отчет" -> ReportsWindow.select_report_file().

Поток:
- QFileDialog выбирает TXT из текущей папки сохранения.
- ReportsWindow.load_report_from_file(filepath).
- TextReportParser.parse_file(filepath).
- TextReportParser.extract_sir_from_filename(filepath).
- TextReportParser.parse_content(content, sir).
- ReportsWindow.apply_loaded_report(metadata, sections).

Что восстанавливается:
- Build, BD, SIR, SQL, DOC, Tax Referens.
- Флаги include_sql, include_doc, include_tax_reference.
- Разделы, номера, заголовки, precondition, scenario, issue_type, issue_text.
- Исполнитель пытается восстановиться из attachment вида attachment:<SIR>_<performer>_<NN>.jpg.

После открытия:
- Секции подгоняются под количество из файла.
- attachment-счетчики сбрасываются и синхронизируются по тексту.
- refresh_live_state() обновляет preview, прогресс и autosave.

11. Attachment-зависимости

Формат:
- attachment:<SIR>_<Исполнитель>_<NN>.jpg

Файлы:
- mod/reports/core/attachment_counter.py.
- mod/reports/ui/reports_window.py.
- mod/reports/ui/section_widget.py.

Логика:
- ReportSectionWidget не генерирует attachment сам.
- Он вызывает attachment_provider, переданный из ReportsWindow.
- ReportsWindow.build_attachment_text() резервирует следующий номер.
- ReportsWindow.preview_attachment_text() показывает следующий номер без резервирования.
- find_max_attachment_index() ищет уже вставленные attachment во всех текстовых блоках секций.
- Ключ счетчика: SIR и Исполнитель в casefold().

Поведение при пустых полях:
- Нет SIR -> attachment:
- Есть SIR, но нет Исполнителя -> attachment:<SIR>
- Есть оба поля -> attachment:<SIR>_<Исполнитель>_<NN>.jpg

12. Autosave-кеш

Путь:
- ReportsWindow.get_session_cache_path().
- По умолчанию reports/.report_builder_session_cache.json.

Что сохраняется:
- version.
- updated_at.
- metadata.
- output_dir.
- sections.
- attachment_counters.

Когда сохраняется:
- refresh_live_state().
- closeEvent().
- save_report().

Когда восстанавливается:
- В __init__ после _build_ui(), _connect_live_updates() и add_section().

Сброс:
- reset_autosave_draft() удаляет cache и tmp, чистит поля, оставляет один раздел.

13. Валидация и прогресс

Файлы:
- mod/reports/ui/reports_window.py.
- mod/reports/dialogs/validation_issues.py.
- mod/reports/utils/widget_helpers.py.

Обязательные верхние поля:
- Build.
- BD.
- SIR.
- SQL, если чекбокс SQL включен.
- DOC, если чекбокс DOC включен.
- Tax Referens, если чекбокс Tax Referens включен.

Обычный раздел требует:
- номер ошибки числом;
- название;
- Pre-Condition;
- Scenario;
- issue_text.

Текстовый раздел требует:
- только issue_text.

Attachment-валидация:
- Любая строка с attachment: должна иметь значение после двоеточия.

Сохранение:
- Без SIR save_report() блокируется сразу.
- Остальные проблемы показываются через ValidationIssuesDialog.
- Перед сохранением обычных разделов дополнительно показывается подтверждение номеров ошибок.
